Redis Connection refused with docker container in Spring boot 3

Spring boot 3 以 Container 連線 Redis 時報出 Connection refused

Nov 03, 2023
2 minute read

情境

先說結論，是 yml 格式問題，因 Spring boot 3x 的格式與 2x 不同導致。

承接上篇的 Seed Project，在做 login validation 的時候若成功驗證用戶資料，則生成 Token 並將 User 資訊儲存在 Redis 中。

完成開發後利用 CI 將所有專案打包成 container，存在 Harbor，並利用 dokcer-compose 拉取，於本地進行開發測試。

如此，藉由設定 host.docker.internal 能讓 container 連接宿主機的 localhost 服務，而使用 IDE 開發時，則直接連接 localhost。

弔詭地就來了，當啟動 docker-compose 後，連線到 MS-SQL 是正常的 (MS-SQL 也用 docker 啟動)，但當要將資料放入 Redis 時卻出現 ConnectException: Connection refused。

解析

由於 IDE 可以但 container 不行，一開始花比較多心力去盤查 docker 網路的設定，甚至有一度懷疑是 Redis 不使用 localhost 在連線時會需要設置密碼。

最後在看官方文件的時候發現 yml 不太一樣，更改後打包 image 竟然就連線成功了…

解法

將 yml 從 spring.redis.host 更改為 spring.data.redis.host 即可正常連線。

1
2
3
4
5

spring:
  data:
    redis:
      host: ${CONTAINER_HOST:localhost}
      port: 6379

我就納悶了，既然打包成 image 後不能連線，那應該在 IDE 連線的時候就要失敗才對啊…
