Spring Boot + MSSQL Quick Starter (Windows)

在 Windows 下快速啟動一個 Spring boot + MSSQL 測試專案

Jul 01, 2023
1 minute read

前言

今天帶大家快速構建一個 Spring boot 專案，並且用 Docker 在 Windows 上面啟動一個測試用的 MySQL，並且用 DBeaver IDE 連線。

近期公司的新人問我怎麼會這麼多東西，記得前公司的協理在帶我的時候說 Always get your hand dirty，不過我得幫他補一句 While you are not Manager … OK 扯遠了。

最主要是針對一些問題我們要能快速的測試，而測試勢必要有環境，如做一些 Container 或 Middleware 的測試要有台 Linux VM。而一些 API 或功能的測試通常也會需要一個測試專案，剛好最近要來二刷跟實作 Spring Security，就來重新建置與紀錄。

建立 MS SQL Container

首先確保你的電腦上已安裝並啟動了 Docker Desktop，安裝方法這裡就不贅述了。

接著打開 cmd，輸入:

1

docker run -e ACCEPT_EULA=Y -e MSSQL_PID="Developer" -e MSSQL_SA_PASSWORD="Passw0rd123" -e MSSQL_TCP_PORT=1433 -p 1433:1433 -d mcr.microsoft.com/mssql/server:2022-latest

輸入後會開始拉取 images，並且啟動，關於 container 的狀態可以在 Docker Desktop 裡面觀察。

DBeaber 建立連線

在 DBeaber 內點選新建連結 (插頭跟 + 號的 icon)，選擇 SQL Server，設定都留預設，要注意是 Host 選 localhost，Database/Schema 留預設 Master。帳號輸入 sa，密碼則是上面啟動 container 時的密碼。

MS SQL 用 container 會預設建立 sa 這個用戶，連線後可以再建立新的用戶與 policy。紀硯連線後點選 Master，在裡面右鍵點選建立新的 DB，這裡我命名為 JAVA_TEST。

	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter-web</artifactId>
</dependency>

application.properties

這裡就設定 spring boot 與 mssql 基本的連線資訊

1
2
3
4
5
6

spring.datasource.url=jdbc:sqlserver://localhost:1433;DatabaseName=JAVA_TEST;encrypt=false;
spring.datasource.username=sa
spring.datasource.password=Passw0rd123

spring.jpa.hibernate.ddl-auto= update
spring.jpa.hibernate.dialect: org.hibernate.dialect.SQLServer2012Dialect

關於 encrypt=false; 是 JDBC 10.2 版本後會預設 Default encrypt to true，若沒有加會導致啟動的時候爆出錯誤(我怎麼會知道呢?)，詳情可以看文件:

1

com.microsoft.sqlserver.jdbc.SQLServerException: 驅動程式無法使用安全通訊端層 (SSL) 加密建立與 SQL Server 的安全連接。錯誤: "PKIX path building failed: sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target"。

基本上就可以啟動專案測試看看連線狀態了，下一步就能逐一去實踐 Repository 的 API 了。
