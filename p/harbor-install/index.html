前言

承上次安裝 Gitlab 後，我思考了幾天，決定還是重新從 Spring cloud 下手，並著手打造整個 DevOps 環境

這邊也先列出幾個前言:

  • 這次 Harbor 使用 Docker-compose 搭建，一樣未來有機會再建到 K8s 內
  • Harbor 憑證生成在 VM 中，需要在 VM 內部配置，同時也要壤他能夠被 Gitlab 與 Services 的 VM 信任
  • Harbor 安裝在 Fedora 上， VM 安裝部分依然跳過，這裡一樣假定我的 Gitlab host VM 的 IP 是 192.168.123.123，Harbor 是 192.168.123.124

生成自簽憑證

下一步要生成自簽憑證

# 創建 ssl 資料夾
cd /opt
sudo mkdir harbor-cert
cd harbor-cert
# 安裝 openssl 並生成憑證
sudo dnf install openssl
sudo openssl req -x509 -nodes -days 3650 -newkey rsa:2048 -keyout my-harbor.key -out my-harbor.crt

設置 windows host

由於是內部的系統，部分的 Devops 設定就不會暴露到 server 外部瀏覽，在正式的 production 情況下可能會是親臨機房或是 VPN 連線，而這裡就以遠端連線的方式到 windows server 內就好。

由於是自簽憑證，需要在 windows server 的 hosts 檔案中把 ip 和 FQDN 寫進去，hosts 路徑在 C:\Windows\System32\drivers\etc

在裡面增加:

1

192.168.123.124 my-harbor

之後就能夠在 windows server 內使用瀏覽器訪問 https://my-harbor 啦!

預設的帳號密碼是

  • admin
  • Harbor12345

記得務必養成習慣，安裝好後立刻改密碼
