Gitlab CI, The kernel does not support PPP

Gitlab CI 使用 openfortivpn ppp not support

May 01, 2022
2 minute read

情境

最近公司使用 Gitlab-CI 進行 DevOps Pipeline，由於 Gitlab 架設在法國，在 pipeline trigger 時，想讓其中一個 job 能夠連線到測試環境的 Staging Server 中傳遞一些參數。

流程上要先建立 VPN 連線，在使用 ssh 連線。

測試用的 gitlab-cu.yml 如下，Key 則是儲存在 Gitlab Variable 之中，在正式環境其實 sshpass 和 openfortivpn 已經預先安裝到 image 內了:

    - echo "sleep start"

    = sshpass -p "$STAG_USER_PW" ssh -o stricthostkeychecking=no "$STAG_USER"@"INTER_IP" "pwd"

分析問題

pppd: The kernel does not support PPP 說明在主機上並沒有支援 ppp (Point-to-Point Protocol)，而實際上核心的原因是 Gitlab runner 在被建立時，就規範由這個 Runner 啟動的 Container 不具備特權 (Priviledge)，因此 無法使用主機的硬體，我先前還在 container 上不斷嘗試安裝各種 PPP 的東西…

一個解法就是修改 gitlab-runner 的 config.toml 配置檔，或是註冊一個新的 Runner 並給予特權

解決方法

1
2
3
4
5

# Ubuntu 的畫路徑通常在 etc/gitlab-runner/config.toml
sudo vim /etc/gitlab-runner/config.toml

# 將 privileged 改為 true
privileged = true

心得

但是法國那邊不允許打開特權，申請一個新的 Runner 手續又繁雜，還在協調中…
