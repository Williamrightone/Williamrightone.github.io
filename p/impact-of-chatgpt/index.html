The Impact of ChatGPT on Software Engineer

ChatGPT 對軟體業影響的看法

May 01, 2023
1 minute read

先說結論，ChatGPT 是非常強大的工具，想要不被 AI 淘汰，勢必要能做到 AI 做不到的事情，便是需求分析、邏輯陳述與創新。

自從去年 ChatGPT 問世，有不少朋友和我討論這點，當時最令我震驚的是如果我能夠完整地闡述軟體需求與規格，他竟然能夠高效率的做出一支可以使用的 API

有興趣的可以嘗試複製下列規格問 ChatGPT:

使用 Java 11 搭配 Spring boot 2.5.7 版本，建立一個 Web MVC API，要有 Controller，Service，ServiceImpl，Repository。Repository 使用 Spring Data JPA，配合 Lombok 簡化 Getter Setter 設置。Data Base 使用 SQL Server。

這支 API 是用於計算 BMI。會以 POST 傳入 json 格式，包含 User 的 ID，身高(公分)，體重(公斤) 三個參數。先在 Controller 接受，由 Service 這個 interface 定義方法，ServiceImpl 實作 Service，並注入 Repository，將此 Json 存入 DataBase 的 BMI table 內。

請提供我 pom.xml, Controller，Service，ServiceImpl，Repository，Entity 與 User 類別作為 Dto。

結果 ChatGPT 成功地寫出了一隻可以計算 BMI 並存入 DB 的 API，雖然他將算 BMI 的方法寫在 Controller 內部，但可想而知若我給予更完整的規格，並且逐步調整與完善，ChatGPT 一定能夠完成大部分碼農的工作。

這時值得反思的就來了，如果提供 ChatGPT 完整的需求規格以及架構，就可以生成軟體程式碼的話:

  • 未來僅需要 BA、SA、SD 就能完成大部分的程式生成
  • Senior Review AI Code
  • 新手工程師入行的門檻會隨之提升
  • 如何讓 AI 寫出更好的 Code 會是下一個軟體需求
