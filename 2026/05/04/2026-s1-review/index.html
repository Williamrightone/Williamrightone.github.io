2026 第一季 Review

回顧了一下, 2026 年第一季產出好像有一點高… 加上這一篇竟然有 12 篇文章, 其中 Development 占了 7 篇, Design Pattern 2 篇, Life 3 篇; 平均一個月能寫三篇文章, 連我自己都驚訝。

Redesign 文章

這季 Redesign 的相關文章偏多, 一方面是想趁自己忘記細節之前把之前遇過的奇人軼事給寫出來, 同時以前的同事 K 老弟也奉獻了一篇文章的想法。

巧合的是四月在靠北工程師上恰逢 Redesign 之亂, 我腦袋內建的 k-means 分出了三派的 Persona。

  1. 靠北半桶水的新人(不僅是剛出社會, 也可能是剛 on board) 眼高手低, 為了重構重構
  2. 強調重構的 trade-off, 核心目的與商業邏輯
  3. 純戲謔

而我自己倒是想開闢另外一條賽道, 就是:

當你忍不住想重構程式時, 需要重構的是團隊作法, 而不是程式本身

在我撰寫的 關於 api retry 的實作方法 和 代理掛號系統的流程 Redesign 都有強調一個問題, 就是:

到底這個程式是如何誕生的?

換個方式講, 如果可以, 如果知道, 誰不想寫出漂亮的程式碼? 我這裡的漂亮有明確的定義, 好理解 & 好維護。

因為一個正常的專案勢必須要:

  • 多人協作, 多人交接維護
  • 部署並消耗實體資源

因此大家才會期望程式好閱讀好理解, 效能正常(不要打爆 DB, 避免 N+1, 避免巢狀結構).

這裡也爆料 K 老弟當年做過的奇葩設計, 我已經取得他的同意公開了 XD:

記得當時為了讓每一個請求可以放入 Client Request ID (idempotent key), 他竟然在 JWT Filter 內把 servlet request 的 json 轉成 string 拆開, 把 header 內的 key 塞入後再 append 大括號回去, 他還笑咪咪的和我說這招叫做橫刀奪愛 (獵人看太多), 然後和我們說記得 request 要 extends BaseIdeReq, 就能拿到資料; 其實明明一個 ThreadLocal 就可以解決的事情, 他卻做了這一波騷操作, 讓我哭笑不得。

你說 K 老弟不認真或沒能力嗎? 在那個還沒有 AI 的年代, 他靠著 google 和翻林信良老師的 servlet 書籍 (看來是沒有看到 ThreadLocal 那段), 用自己的方式解決了問題。

當然這個問題在 Code Review 時就被我們擋掉了, 且對我們彼此來說也是非常寶貴的一課。

這時就能發現, 詭異的程式在有制度的環境下, 幾乎能夠被攔截掉, 而專案最終會走向需要重構, 多半不是因為能力不足，而是因為現實條件往往讓人感到沮喪:

  • 是否團隊沒有 Senior 可以協助 Review 或討論, 大家菜雞互啄
  • 是否需求與規格文件模糊，第三方行為全靠通靈
  • 是否先有結果才有測試案例
  • 是否開發時間被壓縮到先能動再說
  • 是否到最後工程師只保障自己的需求, 其他的不管

到最後就發現:

程式只是在這個環境下必然的產物, 今天就算在合理 trade-off 內成功重構了程式, 若團隊依然是同樣的風氣, 那這種程式遲早會再度累積成大家不能接受想重構的樣子。

讀懂底層邏輯之後, 就會發現討論重構很容易陷入一個假議題的窠臼, 如果環境允許, 開發時間充足, 有專案業的架構師與分析師, 誰會故意去寫出需要重構的程式碼? 若你有相關的需求文件可以看, 你何必要努力去猜這段程式在寫什麼?

如果不解決這些問題, 那只談重構有何意義?

然而現實就是如此骨感, 成本才是一切考量的根源, 而我們唯一能做的也就是兢兢業業, 確保自己在這個環境下也能做到最好的樣子。

AI & Side Project

這個四月我訂閱了 Claude Code Max, 在 Vibe Code 開發上有大幅的感受, 雖然還沒有引入 Github 上優化 AI 的專案, 但還是能感受到 AI 帶來了便利,我也整理在關於 Vibe Code 開發心得與架構規範這篇文章內.

靠著 AI 我開發了 VoxelMD 和 Lumo Tarot 這兩個 Web 服務, 也把之前的 Casha Pos 用 AI 生成 Prototype 放到 Github Page 上, 說真的, 有完全提升我的開發樂趣, 原本只想訂閱一個月, 結果隨著點子慢慢浮現, 只好再訂一個月了.

我們偉大的黃總也說, 他期望一個高薪的軟體工程師投入約一半的薪水在 AI 的使用上, 此言還真不假。

同時我也靠著與 AI 相互學習, 手動搭建了一台本地的 K8S (終於脫離 Kubeadm 啦), 之前做 Devops 的時候比較少機會碰到這些配置, 在自學的時候也是一堆疑問, 現在一步一步做起來還真的挺踏實的。

今年的主力文章應該也會靠著 VoxelMD 和其他的專案心得築起吧~。

跳舞與生活

這季除了去平溪看天燈以外, 我在年後加入了 The Moon Hoppers 舞團, 團員幾乎是我在跳 Swing 路上的老師, 身為一個超級菜鳥, 能和他們一起練習進步真的非常榮幸 (但壓力也是大到爆), 記得團練的第一天, 練習了一首又快又難的歌, 我實在有些跟不上, 但是到後面大家輪流跳出來 show 的時候, 心中真的有一種莫名的悸動, 我未來也會變得和他們一樣, 我也正在和他們一樣。

四月初青梅成熟, 好友邀請我到他家釀梅酒, 記得大學的時候曾在酒吧裡打工, 出社會後還有去參加過釀啤酒的 workshop, 做泡酒還是第一次。

四月底的時候忽然看到 Mary See the Future 居然開了一個專場, 實在不可思議, 身為一個聽了十幾年的老粉 (見證過主唱在小河岸怒噴核電歸零 & 麥克風飛到二樓), 由於前兩次表演的時候有發現他們用了更多的 Program, 也把許多歌從 full band 配置更新成 Program 配置了, 個人還是更喜歡 full band 配置, 但為了我的青春也是手刀購票支持一下.

結果這場還是挺驚喜, 這幾位有在復健了 希望未來還能聽到他們的創作, 還能夠體會到當時第一次在小河岸聽最難寫的歌時, 那一種對音樂的熱情與渴望(那一晚我到 StreetVoice 找 Josh 的帳號, 裡面的 Rough for Rough 真的都是寶藏)。

在三月與五月初也分別參加了兩個在台北的大型舞會 Spring Hop 和 Bubble Pop(就昨天啦), 大型舞會通常都會邀請國外的舞者當評審與嘉賓, 我也見到了一位很喜歡的韓國舞者 Woody Kim, 在一年多前起我就很常看他的 youtube 影片, copy 一些動作與 flow 做練習, 然後韓國歐巴也是真的挺帥, 有合到照, 拍照時我整個小迷弟, 一直說我有追蹤他的 youtube, 有看他在 Savoy Cup 超帥的動作; 他也說剛才有看我比賽, 覺得我跳很好(他是評審), 追星成功。

在 Bubble Pop 的第一天我參加 City Cup Rookie (舞鈴 1~3 年者) 比賽, 獲得第二名, 第一名還是我的同學也叫威廉. 這次台北是 Asian City Cup 的第一站, 還有韓國, 泰國站, 最後在今年 11 月回到台北總決賽, 看看到時有沒有機會扳回一城。

結語

除了工作之餘, 我還是希望能夠把生活的一些內容和方法, 點子應用到軟體內, 讓他們變成興趣也算是一件好事, 剛好又吃到了 AI 的紅利, 讓我從侏儒一下變身成巨人。

在生活上, 有發現其實舞越跳越多, 膝蓋也慢慢有些狀況, 這時我也思考如果有一天我不跳舞了, 會不會生活就失去一大重心? 以前總想著 Work Life Balance, 結果現在到是變成了 Life Life Balance 了。

最後這一季(雖然是四個月) Review 我覺得算繳出了一張不錯的成績單, 希望下一季度能夠過得更好, 也祝福讀到這裡的朋友今年心想事成。
