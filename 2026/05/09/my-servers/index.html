那些年, 我組過的伺服器

由於我昨天又買了一台電腦, 現在房間裡共有八台, 被我的朋友們瘋狂嘲笑:

電腦比前女友多

所以我決定來介紹一下我組過的電腦和用途…

故事要從 2018 年講起. 我辭去新竹電子廠工程師的工作, 到台北念碩士, 開啟了我的軟體之路. 當時各種影音平台, Youtuber, 視頻博主都挺流行撿垃圾組電腦, 像是 Huan, 翼王, 極客灣等等, 什麼幾百幾千組一台電腦的影片一大堆, 我也是看不膩.

那時剛好同樣在台北念碩士的哥哥, 有一台 i5 6500 + GTX 960 小鳳凰沒在用, 就給我抱回實驗室玩. 結果小鳳凰的機殼氧化, 我又買了一套小尖兵機殼在實驗室裡換裝, 拆拆裝裝被教授看到, 老闆和我說:

威廉原來你會動這些電腦, 那你把我們實驗室的電腦處理報廢和升級一下

我看了老師給的那可憐的預算, 心想你這是把我當耶穌, 手摸一摸礦泉水變葡萄酒, i3 變 i9… 雖然心裡碎念著, 但還是幫忙把實驗室的電腦都報廢和整理了一番, 也就是那段時間, 我心裡有某種開關似乎被打開了.

後來畢業進入 SI 公司, 軟體部的協理(也是我的貴人) 送我去受 Azure 培訓之前, 知道我對 Infra 的經驗幾乎是 0, 便丟了一台 ASUS 的桌機給我, 讓我在上面的 windows server 內安裝幾台 Fedora 34 做內部開發的 middleware 集成. 當時 linux 根本沒碰過幾次, vim 進去還出不來(但是我當時知道怎麼置中一個 div), 摸爬滾打兩周後總算是建起了 sql-server, Jenkins, mq & Harbor.

也就是這個成功, 正式開始了我瘋狂購買電腦之路.

1. IT 鐵人賽起手式, Dell 7040m *2

因為有了搭建 Fedora 的經驗, 我開始嘗試在家裡打造類似的 pipeline, 先後購買了兩台 Dell 7040m 做 DevOps Server, 在上面搭建了 Harbor, Jenkins 等服務, 然後還參加了 ITMan 鐵人賽 DevOps 組(因為有寫 Angular, 當時還被 will 保哥私訊…), 一台作為 DevOps 機器, 另一台則作為 Production 機器.

而這個專案也是我的 Colorly 服務原型.

鐵人賽結束後我把其中一台放在公司裡架設 gitbook, 當時我只是一個菜鳥 SA 兼任 PG, 因為沒用 https, 又開一堆 port, 被我的 SD 廣播到, 寫了一個腳本打爆我的服務和 gitlab, 然後順著 ip 找到我的位置…

這兩台機器陪了我很長一段時間, 其中一台在我組 X99 平台後賣掉, 另一台現在仍在我的房間內.

2. 第一台 NAS

在大學的時候我就會用小蟻 (小米的運動相機) 記錄生活, 後來入手了 insta360 go, 開始有很多影片儲存的需求, 此時我便土炮組了第一台 NAS, 灌的還是 TrueNas. 也是被司波圖這個大陸博主燒到, 我覺得我需要這個酷東西.

但礙於預算限制, 我買了一台帶電源的機殼, 並把光碟區和 io 都換成了硬碟籠, 真的是 潮到爆 很土, 後來使用了一段時間, 當時的女朋友不喜歡風扇聲(他是兩顆小風扇), 只好賣了.

3. X99 初登場

在 SI 工作的一段時間, 我去支援台灣交通電子支付的案子, 過程中接觸到了一套 Infra 方案, 便是在 windows server 上面搭建 VM cluster, 並利用多台 windows server 相互做備援. 而這套做法也被我沿用至今, 若有看過我的 VoxelMD, Casha Pos, Colorly 的文件應該都不陌生.

大家還記得我哥的那台小尖兵吧, 為了能模擬出生產的環境, 我一腳踏入了洋垃圾的世界, 買了一片華南的 X99, E5-2628LV4 12 核 24 緒放進去, 框框數起來, DDR4 那時白菜價, 直接插好插滿, 開了好幾台 VM. 在內網環境下, 我有了一台真正意義上的家用伺服器.

4. 蝸牛星際

大家看我的第一張照片應該有注意到, 其實我有一個 ORICO 五盤位的 3.5 吋外接硬碟盒, 但我覺得還要接電腦太 low 太不優雅了, 最後我不顧當時女友反對(她覺得很佔空間), 直接淘寶了一台蝸牛星際 6 bay, 用的還是 AMD Sempron 3850 集成板, 外加 pcie 轉 Sata 擴展卡組成. 光看那個熱插拔盒我就覺得我又提升一個檔次.

而就在前幾個月, 我把他賣了, 因為我用了 Gemini 送 2T 的空間, 以及單身後生活空間變小的原因.

5. X99 雙路

後來我進入法商, 開始學習 K8s, 我極度不滿足於 Kubeadm 那種小兒科, 覺得一定要一個 Master, 三個 Slaves(現在要叫 Worker 啦!) 才夠屌! 於是我轉頭把小尖兵賣了, 然後購入 X99 雙入, E5-2683 v4 *2, 32 核 64 線程!!! 真正的數框框。

Ram 更是來到驚人的 DDR4 ECC 16 * 8, 主機板還是 E-ATX, 整台電腦大到令人頭痛, 我在光華買摩托車還載不動. 至於小尖兵賣給了一個很詭異的先生, 當時我的主力 T470p 壞掉, 只好購入 RTX 3050 的筆電, 便把 X99 和之前從小尖兵裡挖出來零件組成另外一台電腦一起賣了, 那位先生說要給自己兩個小孩用, 他不是很在意規格, 只問我能不能滿足文書等等需求, 我解釋一堆東西他反而不是很感興趣.

而這台 X99 雙路也讓我實現本地一條龍的服務, 基本上已經把工作接觸到的做法給複現出來了, gitlab + devops + spring cloud 內網環境完美組合. 面試的時候直接 teamViewer 到家裡的筆電(我當時還有一個 Surface pro…), 然後遠端桌面連線過去展示, Harbor, Jenkins, SonarQube 只能由內網訪問, windows server 再利用內網穿透與 nginx 反向代理, 模擬 tier 1 tier 2 的環境.

Dev Server 其實有兩台, 內部部署多個 instance 做備援; Devosp 也升級為 DevSecOps, 要先經過 Quality Gate, Dependency Track, trivy 等測試才有辦法 merge, 每個 stage merge 後都可以 docker-compose pull 下來開發與測試.

其實這也就是台灣金融業很愛使用的地端方案。

看到這裡, 你可能覺得這應該是我在伺服器上的終點了?

大錯特錯, 我當時分手, 趕著搬家被迫住到一間一度電 5.5 元的頂樓加蓋, 面試的那個月我的電費來到驚人的 3000 多!

當晚我把 X99 關機拔線, 面對換工作的壓力與分手的痛苦, 我抓著帳單躺在地上大哭, 覺得憑什麼認真的人要受到懲罰, 雖然這些錢並不會對我的生活造成多大的影響, 但仍然成為當時壓垮我內心防線的最後一根稻草.

6. Mac Mini & asRock 時代

被電費狠狠揍了一拳後, 我開始考慮多個省電的方案.

首先將 devops 的責任從 X99 VM 搬到 Dell 7040m 上, 並開始嘗試 github private runner, 效果其實挺理想的, 但是 private runner 時常斷線, 我甚至要手動寫腳本去啟用他.

加入 IBM 後, 我首次使用 M 系列的 Macbook pro, 當下真的有驚豔到 — M2 16 GB 的配置跟我的 windows 32GB 配置在處理微服務專案上竟然體感差不多, 表示 mac 的優化是真的做得很好. 於是我動手買了一台 Mac Mini M2 作為 devops 伺服器, 取代後面的 devops 機器, 同時又在上面跑了一個 8B 的模型 api, 給我後面的服務使用.

隨後我也把 surface pro 賣了, 並買了一台 asRock J3000 做為跳板機使用. 至於原本 X99 雙路上的服務, 我則買了一台 deskmini x300 想嘗試極限部署, 但散熱效果不理想, 就先收著等出售, 而那些 side project 也只能先下下來, 或是留著前端 prototype 放在 github page 上.

順便講一個鄰居故事

我搬離頂樓加蓋後, 到一個分租套房內.

就在一個夏季的夜晚 2 點, 我的門外傳來了嚎叫聲, 出去一看是最近搬進來的大學生, 半夜打電動鬼吼鬼叫, 我敲門好幾次, 外加連絡房東, 這孩子總會在幾天後克制不住自己繼續鬼叫.

一怒之下, 我在 asRock J3000 上用 Claude 寫了一個腳本, 深夜定時開 5 個 Thread 瘋狂下載檔案把共用的光纖頻寬吃滿(反正應該只有他半夜會用網路).

小朋友, 斷網囉

只聽他一聲驚呼, 之後安分了很多. 好在他手機沒有辦吃到飽?

以下是示意:

7. 電費 vs 性能的最佳解: AM4 平台

為了處理空間與電費在開發學習上的平衡, 我最終組了一台 AM4 R5 3600X 平台, 有著 128GB 的 DDR4 (現在根本是黃金價格…). 在性能提升的同時, 電費也降低了不少, 唯獨在服務 boot up 的時候 cpu 會有滿載壓力. 現在即便是運行 VoxelMD 這種 K8s + Tracing 的 cluster 也是游刃有餘, 也是我當前正在使用的方案. 另一方面是我看開其實自己還用不到這麼多核心, 不過我依然天天在觀望, 考慮當 R7 3700x 降到 2000 元的時候升級一下.

而 X99 雙路則是賣給了一個黃牛小老弟?! 這也是一個奇妙的故事. 出售期間大概是政府普發 1W 的時候, 一個幫別人跑搶票系統的小兄弟向我購入了那台 X99, 似乎是給他程式的人說我那台可以買. 當時記憶體已經在瘋漲, 但我並沒有隨之起舞, ECC DDR4 就按照當時購入的價格售出. 他也和我小聊了一下, 原來做黃牛還有組織團夥, 負責寫程式與設置 ip, 以及操作的人, 我之前也有寫過搶票程式, 但都是自己使用, 從來沒想過原來還有組織架構, 算是一次很不一樣的經驗.

這時的 CI/CD 就演化成將 DevOps Machine 獨立出來(在 Dell 7040m 上) 並使用 Github Private Agent 在內網環境作部署.

8. HP Z2 Mini G5

然而就在這周, 我把 VoxelMD 上線了, 卻遇到尷尬的 TTS 效率問題. 3 分鐘的音檔用 CPU 跑大約要花 100 秒左右, 改用筆電 GPU RTX 3050 4GB 跑, 只要 9 秒就搞定, 搞得我當下開蝦皮去看顯示卡.

不過 RTX 3050 以上 4 ~ 8GB 的顯示卡二手價格都落在 5000 - 8000 之間, 實在有點買不下去.

但就在這時, 我看到了 HP Z2 Mini G5, 搭載 W-1250 & Nvidia T1000 顯示卡, 只要一萬. 我印象中 T1000 是專業繪圖卡, 4GB 也不便宜, W-1250 6C12T 待機功浩也大概在 10W ~ 15W, 似乎, 可以, 考慮…

另外是隨著 X99 的售出, Casha Pos 和 Colorly 都下線了一段時間, 考量到想長期讓他們都在線, 我其實已經在著手把這些微服務專案轉換為單體部署(這些會寫成另外一篇文章), 預計部署到目前閒置的 7040m 內, 不過散熱與噪音都還在評估中. 同時 TTS 的問題也還沒解決, 此時 Mini G5 就像緣分到了一樣, 直接跟我在轉角相遇.

當我回過神, 我已經下單了…

結語

就這樣, 我鬼迷心竅的又買了一台電腦, 在我打到這篇的時候, 他已經到貨了, 我準備在這個母親節前重新把單體版本的 VoxelMD 搭建上去, 之後慢慢維護與更新. 這樣我常駐開機的電腦就只剩開發筆電, Mac Mini M2 做 devops 和這台 G5 當 production, AM4 則是需要面試或測試學習研究 Cluster 的時候可以開個一兩周, 我想這就是我目前最佳的方案吧.

回頭看, 從那台被前女友扔掉的雷姆 7040m, 到這台準備領貨拆封的 G5, 中間倒掉的機器比女朋友多, 倒掉的電費比薪水單還顯眼. 但這些機器確實陪我從 vim 出不來走到能在家裡跑 K8s cluster 模擬 tier 1/tier 2 環境跑面試, 從 SI 到外商, 對我來說, 與每一台電腦的時光, 都有著獨特的回憶, 總覺得隨著每一次的硬體升級, 自己的人生也進入了下一個階段.

然後我再次強調我不是家裡有礦, 雖然每次買電腦多少都有點衝動消費的元素… 但房間的閒置電腦我也是很努力在販售中… 所以不要再說我有什麼賽博倉鼠病了(喜歡囤積 3C 物).

謝謝你看到這裡, 我要去領貨組電腦了, 預祝大家母親節快樂。
