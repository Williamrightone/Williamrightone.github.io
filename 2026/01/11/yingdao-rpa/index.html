Automa | 影刀 RPA 測試與心得 (撰寫中...)

RPA, 全名 Robotic Process Automation 直議機器人流程自動化, 概念上就是寫一套軟體程式來模擬人類在電腦上的操作行為的技術.

就定義上, 我寫一個 shell script 腳本幫我執行任務, 其實也是在做一種 RPA, 但現在大家更傾向把 RPA 當作是一個一般人也能操作的自動化流程軟體.

早期在法商做後台的時候, 有遇到一些 mainframe 資料只能透過 windows .exe 軟體內操作與複製, 我們的 SRE 就在 vm 內啟用 Task Scheduler 觸發 tagui, 去執行這些操作.

因此, RPA 的存在也比較像流程中的黏合劑, 在企業裡常見這種情況:

  • 老系統沒有 API
  • 第三方平台不能串 (有複雜的驗證操作流程)
  • 內外部系統流程斷裂

所以通常 RPA 都會是在沒有 api 時的最後手段, 就我個人的印象, Deloitte 的內部審計與報表, HSBC 的銀行營運及 KYC 都有使用到 RPA 的技術.

剛好近期有收到一些類似的需求, 又想到過往曾接處過 tagui, 這幾天就來重新模擬測試一下.
